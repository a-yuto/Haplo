# Haplo 言語仕様 — 字句・構文（§3.1〜§3.5）

## §3.1 字句（トークン）

- コメント：行 `-- ...`、ブロック `{- ... -}`（ネスト可）
- 識別子：先頭は文字、以降は文字・数字・`_`。末尾プライム `x'` 可。次元変数 `m, n, k` も通常の識別子
- リテラル：整数 `42` / 浮動小数 `3.14`, `1e-3` / 真偽値 `true`, `false` / テンソル（§3.2）
- 記号類：`+ - * / ^ @ = : -> , ( ) [ ] |>` ほか
- **数式記号は不採用**。集約等は ASCII 名（`sum`, `mean`, `grad`, `transpose` …）

---

## §3.2 テンソルリテラル（B＋D 併用）

小さなリテラルは **B 記法**（列はカンマ、行はセミコロン）：

```
v = [1.0, 2.0, 3.0]              -- Tensor[3]
a = [1.0, 2.0, 3.0;              -- Tensor[2,3]
     4.0, 5.0, 6.0]
```

それ以外・高階テンソルは **D 記法**（shape をリストで渡すコンストラクタ）：

```
zeros [2,3]    ones [3]    eye 3    randn [2,3]
reshape v [2,3]            stack [r0, r1]
```

---

## §3.3 束縛と関数定義（Elm 風・完全不変）

- 束縛：`name = expr`（型注釈は別行 `name : Type`、任意）。**完全不変**＝再代入なし
- **シャドーイング許可**（同名の再束縛は新しい不変の束縛であって再代入ではない）
- 関数定義：`name 引数... = expr`（`fn` も波括弧もなし）。多引数はカリー化
- 呼び出し：空白区切り `predict x w b`（括弧・カンマなし）
- ローカル束縛：`let ... in ...`

```
loss : Tensor[3] -> f32
loss w =
    mse (predict x w 0.0) y
```

---

## §3.4 式と演算子

すべてが式（`if` / `case` / `let..in` も値を返す）。優先順位は**型に依存しない構文規則**で固定。

| 優先 | 演算子 | 結合 | 意味 |
|------|--------|------|------|
| 高 | `f x` `A[i,j]` | — | 適用・添字 |
| | `-x` | — | 単項マイナス |
| | `^` | 右 | べき乗（要素ごと） |
| | `* /` | 左 | 要素積・除算 |
| | `@` | 左 | 行列積 |
| | `+ -` | 左 | 加減（要素ごと） |
| 低 | `== != < <= > >=` | 非結合 | 比較 |

- `+ - * / ^` はテンソル同士なら要素ごと（同 shape 必須）、スカラーは自動ブロードキャスト
- 演算子の「shape 妥当性」は型検査時に決定（優先順位は不変）
- パイプ `|>` を提供

---

## §3.5 制御と反復（for/while なし）

- 条件は式：`if c then a else b`
- パターンマッチ：`case e of ...`
- **ループは持たない**。反復は**再帰**または高階プリミティブで表現
  - `iterate init n f` … `f` を `init` に n 回適用して最終状態を返す
  - `fold xs init f` … 系列に沿って状態を畳む（scan 相当）

```
step : Tensor[3] -> Tensor[3]
step w = w - lr * grad loss w
trained = iterate (zeros [3]) 1000 step
```
