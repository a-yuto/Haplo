## P1 で実装する最小セット（G3 達成に必要）

G3 = 線形回帰サンプル（`docs/spec-types.md` §3.8）が走ること。

1. **`Tape` 構造体**（演算の記録と逆伝播）
   - `eval()` の呼び出し順に演算ノードを記録する
   - 逆向きに辿って各テンソルの勾配を算出する
2. **`grad` 組み込み**：`grad : (Tensor -> f32) -> Tensor -> Tensor`
3. **`iterate` 組み込み**：`iterate : a -> Int -> (a -> a) -> a`
   - または再帰関数で代替（前方参照が実装される前の暫定手段）
4. **前方参照対応**（two-pass スコープ構築）
   - 1 pass 目：名前だけ登録
   - 2 pass 目：本体を評価

ツリーウォーキング選択の恩恵：`eval()` に `&mut Tape` を渡すだけでテープ記録を追加できる。

## サンプル検証で判明した SPEC との差分（`examples/` 実行時）

`examples/*.hpl` を P0 インタプリタで走らせて確認した、SPEC（§3.4）に未達の挙動。
いずれも回避策はあるが、SPEC どおりに書けるようにするための課題。

| 箇所 | 現状 | SPEC との差分 | 対応予定 |
|------|------|--------------|---------|
| スカラー ÷ / ^ テンソル | `Div`/`Pow` はテンソルが左のときのみ対応（`eval_binop`） | §3.4 ではスカラーは自動ブロードキャストのはず。`1.0 / t` が型不一致になり sigmoid 等が書けない | P1〜P2 |
| `1D @ 2D` 行列積 | `@` は 2D×2D / 2D×1D のみ。1D×2D は明示エラー | 行ベクトル（`Tensor[3]`）を重み行列に掛けられない。2D で書く必要あり | P2 |
| パイプ `|>` の改行 | 同一行のみ解釈。改行をまたぐと `Pipe` が予期しないトークン扱い | §3.4 のパイプを複数行に分けて書けない（パーサが改行を区切りと誤認） | P1 |

注: `Add`/`Sub`/`Mul` はスカラー左右どちらのブロードキャストも対応済み。差分は `Div`/`Pow` のみ。

## コーディング規約

- コメントは `//`（非公開コード）または `///`（公開 API）
- テストブロック `#[cfg(test)]` へのコメントは不要
- 外部クレートは最小限（現在 `ndarray` のみ）
- `unwrap()` はテスト内と「到達不能」が証明できる箇所のみ許容
- エラーは `HaploError` に統合（`anyhow`/`thiserror` 不採用、外部依存最小化）

## テスト構成（現在 57 本）

```
cargo test                       # 全テスト
cargo test g0                    # G0 スカラーテスト（interpreter.rs）
cargo test g1                    # G1 テンソルテスト（interpreter.rs）
cargo test integration           # 統合テスト（main.rs）
```

テストは各ソースファイル末尾の `#[cfg(test)]` ブロックにある。
インタプリタのテストが最多（`g0_*` がスカラー 15 本、`g1_*` がテンソル 10 本）。
